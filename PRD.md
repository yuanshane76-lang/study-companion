# 学习陪伴助手 (Study Companion) — 产品需求文档

> 版本：v2.2 | 更新日期：2026-04-13

---

## 1. 产品概述

### 1.1 产品定位

学习陪伴助手是一款面向**自主学习人群**的桌面端学习管理工具。它以「每日学习日报」为核心载体，帮助用户将学习目标拆解为可执行的每日任务，并通过 AI 辅助完成资源推荐、计划调整和学情复盘，形成「输入 → 执行 → 反馈」的学习闭环。

### 1.2 目标用户

- 备考学生（考研、公考、考证等）
- 自驱学习者（技能提升、阅读计划等）
- 需要结构化学习管理的任何人群

### 1.3 核心价值

| 痛点 | 解决方案 |
|------|----------|
| 学习计划难以坚持 | 每日日报驱动，写总结即完成，降低执行门槛 |
| 资源分散、找不到好材料 | AI 推荐学习链接，自动关联学科 |
| 计划赶不上变化 | AI 根据完成情况自动调整后续计划 |
| 缺乏复盘意识 | 周报自动生成，可视化学习轨迹 |


3. **详情页**：具体学科下的日报列表、计划时间线、资源池（含计划编辑/删除入口）

## 3. 功能需求

### 3.1 MVP 范围（两种输入类型）

MVP 阶段仅支持以下两种输入，后续迭代再扩展：

#### 输入类型一：文本目标

- 用户输入一段文本描述学习目标（如"本周完成《算法导论》第 3-5 章"）
- AI 自动拆解为每日学习任务
- 生成对应的每日学习日报模板

#### 输入类型二：PDF 上传

- 用户上传 PDF 文件（教材、讲义、论文等）
- 文件大小限制：50MB
- AI 自动解析 PDF 内容（MVP 仅支持文本型 PDF，扫描件/图片型 PDF 不支持）
- 按章节/知识点拆分为学习单元
- 自动生成学习计划和每日日报
- 解析失败降级：提供手动输入计划内容的入口

### 3.2 每日学习日报

日报是产品的核心交互单元，包含以下模块：

| 模块 | 说明 |
|------|------|
| ⏱️ 学习计时器 | 开始/暂停/停止，自动记录当日学习时长 |
| 📋 今日任务 | 从计划中拆解的当日待办事项 |
| 📚 学习内容 | 根据学科类型展示具体学习内容（见下方详细说明） |
| 🔗 推荐链接 | AI 根据当日学习主题推荐的相关资源 |
| 📝 学习笔记 | 用户自由记录区域（支持 Markdown） |
| ✅ 完成总结 | 写总结 = 标记当日完成（核心完成机制） |

**完成机制**：用户在日报中写下当日学习总结，即视为完成当日任务。总结内容不得少于 10 个字符（含标点），完成后仍可编辑——清空总结即恢复为未完成状态。降低完成门槛，避免因未打卡而产生挫败感。

**学习计时器**：用户可在日报页点击开始计时，计时数据自动归入当日的学习时长统计，支持暂停和停止。停止后计时归零，用户可多次启动。

**学习内容展示规则**：
- 每日生成的 Markdown 文件必须包含当天所有应学习的具体内容
- **单词类学习**：直接展示原词及释义（例如：`abandon - v. 放弃，抛弃`）
- **文章/视频/音频类学习**：用链接替代，点击可跳转至原始资源
- **公式/代码类学习**：直接嵌入公式或代码片段
- **其他不适合直接展示的内容**：统一用链接+简短说明的形式呈现

**示例 - 英语学科日报内容区**：
```markdown
## 今日单词（100个）
1. abandon - v. 放弃，抛弃
2. ability - n. 能力，才能
3. absolute - adj. 绝对的，完全的
...
100. zeal - n. 热情，热忱

## 今日阅读
- [The Benefits of Reading](https://example.com/article1) - 关于阅读益处的科普文章
- [How to Improve Vocabulary](https://example.com/video1) - 词汇提升技巧视频
```

### 3.3 AI 推荐链接

- 根据当日学习主题，AI 自动推荐 3-5 条高质量学习链接
- 链接来源：公开教程、文档、视频、论文等
- 用户可标记链接为「有用 / 无用」，反馈影响后续推荐
- 链接池独立存储，可跨日报复用
- MVP 阶段不校验链接可用性（成本高、体验差），在推荐链接区域添加免责提示："AI 生成的链接可能不可用，请以实际访问为准"

### 3.4 学科自动分类

- 新建学习计划时，AI 自动识别学科归属
- 支持用户手动调整分类
- 默认学科示例：英语、算法、新闻、数学、专业课等
- 用户可自定义学科列表

### 3.5 AI 计划调整

- **触发时机**：用户下次打开 App 时自动检测。若发现过去有未完成日（当天未写总结），则触发 AI 调整逻辑。App 未打开不触发后台定时任务。
- **未完成判定**：当日 24:00 前未写总结，标记为"未完成"
- **调整策略**：
  - 未完成 → 顺延任务至次日，压缩非核心任务
  - 提前完成 → 提前推进后续内容或补充扩展学习
- 用户可手动确认或覆盖 AI 调整

### 3.6 周报生成

- 每周日自动生成（检测逻辑：下次打开 App 时，若发现已过周日且未生成周报，则自动补生成）
- 周报内容：
  - 本周完成率
  - 各学科学习时长分布（来源于学习计时器数据）
  - 关键学习成果
  - 下周计划预览
  - AI 学习建议
- 支持导出为 Markdown / PDF

## 4. 数据结构

### 4.1 核心实体

```
Subject (学科)
├── id: string
├── name: string
├── icon: string
├── createdAt: datetime
└── plans: Plan[]

Plan (学习计划)
├── id: string
├── subjectId: string
├── title: string
├── sourceType: "text" | "pdf"          ← MVP 两种输入
├── sourceContent: string | PDFRef
├── startDate: date
├── endDate: date
├── status: "active" | "completed" | "paused"
└── dailyReports: DailyReport[]

PDFRef (PDF 引用)
├── filePath: string    ← 应用数据目录 pdfs/ 下的文件路径
├── fileName: string    ← 原始文件名
├── fileSize: number    ← 文件大小（字节）
├── pageCount: number   ← PDF 总页数
└── parseStatus: "pending" | "parsed" | "failed"  ← 解析状态

DailyReport (每日日报)
├── id: string
├── planId: string
├── date: date
├── tasks: Task[]
├── contentItems: ContentItem[]           ← 当日学习内容（单词、链接、公式等）
├── linkIds: string[]   ← 引用 LinkPool 中的链接 ID
├── notes: string (Markdown)
├── summary: string | null              ← 有值 = 已完成（≥10字符）
├── completedAt: datetime | null
├── studyDuration: number               ← 当日学习总时长（秒），由计时器累计

Task (任务)
├── id: string
├── title: string
├── description: string
├── status: "pending" | "done" | "skipped"
└── order: number

ContentItem (学习内容项)
├── id: string
├── type: "word" | "article" | "video" | "audio" | "formula" | "code" | "other"
├── title: string                        ← 标题（单词原词、文章标题等）
├── content: string                      ← 具体内容（释义、公式、代码等）
├── url: string | null                   ← 外部资源链接（文章/视频/音频等）
├── order: number                        ← 展示顺序
└── metadata: object | null              ← 额外信息（如单词的词性、难度等）

Link (推荐链接) — 独立存储于 LinkPool
├── id: string
├── url: string
├── title: string
├── source: string
├── feedback: "useful" | "useless" | null
└── createdAt: datetime

WeeklyReport (周报)
├── id: string
├── subjectId: string
├── weekStart: date      ← 周一日期
├── weekEnd: date        ← 周日日期
├── completionRate: number  ← 完成率（百分比）
├── subjectDuration: { subjectId: number }  ← 各学科学习时长（秒）
├── achievements: string[]  ← 关键学习成果
├── nextWeekPlan: string    ← 下周计划预览
├── aiAdvice: string        ← AI 学习建议
├── rawText: string         ← 完整周报文本（支持导出）
├── exportedAs: ("markdown" | "pdf")[]  ← 已导出的格式
└── generatedAt: datetime

LinkPool (推荐链接池) — 全局存储
├── links: Link[]          ← 所有推荐链接，跨日报复用
└── reportsUsingLink: { linkId: string[], reportIds: string[] }  ← 多对多映射
```

### 4.2 存储方案

- 本地 SQLite 数据库存储所有结构化数据
- PDF 文件存储在应用数据目录下的 `pdfs/` 文件夹
- 用户配置（API Key、学科列表等）存储在 `config.json`

## 5. AI 接入设计

### 5.1 API Key 管理

- 用户在设置页自行配置 API Key
- 支持多种大模型后端：
  - OpenAI（GPT-4o / GPT-4o-mini）
  - 国产模型（DeepSeek / 智谱 / 月之暗面等）
- 统一使用 OpenAI 兼容接口格式
- 用户可自定义 Base URL（适配第三方中转服务）

### 5.2 AI 功能调用点

| 功能 | 触发时机 | 输入 | 输出 |
|------|----------|------|------|
| 目标拆解 | 新建文本目标 | 目标文本 | 每日任务列表 |
| PDF 解析 | 上传 PDF | PDF 文件内容 | 章节拆分 + 任务列表 |
| 生成学习内容 | 生成日报时 | 当日任务主题 + 学习计划 | 具体学习内容（单词/链接/公式等） |
| 推荐链接 | 生成日报时 | 当日学习主题 | 3-5 条链接 |
| 计划调整 | 检测完成偏差 | 完成情况 + 剩余计划 | 调整后的计划 |
| 周报生成 | 每周日（下次打开时补生成） | 本周所有日报 | 周报文本 |
| 学科识别 | 新建计划时 | 计划标题/内容 | 学科分类 |

### 5.3 AI 降级策略

- 无网络时隐藏 AI 功能入口（推荐链接、计划调整、周报生成都不可用）
- 基础功能（查看日报、写笔记、编辑计划、计时）完全离线可用
- AI 调用超时 30s 后返回降级提示，不影响用户继续操作

## 7. 交互流程

### 7.1 核心流程：文本目标输入

```
用户输入目标文本
    ↓
AI 拆解为每日任务
    ↓
自动识别学科 → 归入对应学科
    ↓
生成每日日报模板
    ↓
AI 生成当日学习内容（单词/链接/公式等）
    ↓
每日打开日报 → 查看任务 + 学习内容 → 学习
    ↓
写学习笔记 → 写完成总结 → 标记完成
    ↓
AI 检测完成情况 → 自动调整后续计划
```

### 7.2 核心流程：PDF 上传

```
用户上传 PDF（≤50MB）
    ↓
AI 解析 PDF 内容
    ↓
解析失败？ → 是 → 降级为手动录入计划
    ↓ 否
按章节拆分学习单元
    ↓
生成学习计划 + 每日日报
    ↓
AI 生成当日学习内容（从 PDF 提取关键点/公式等）
    ↓
（后续流程同上）
```

## 8. 非功能需求

| 项目 | 要求 |
|------|------|
| 启动速度 | 冷启动 < 3s |
| 本地存储 | 所有数据本地存储，不上传云端 |
| 数据安全 | API Key 加密存储 |
| 离线可用 | 基础功能（查看日报、写笔记、计时）离线可用 |
| AI 降级 | 无网络时隐藏 AI 功能，不影响基础使用 |
| PDF 限制 | 文件 ≤ 50MB，仅支持文本型 PDF |
| 学习内容生成 | AI 生成每日学习内容的响应时间 < 10s（100个单词以内） |
| Markdown 导出 | 日报内容可完整导出为标准 Markdown 格式，保持排版 |
| 离线缓存 | 已生成的学习内容缓存到本地，无网络时仍可查看历史日报 |
